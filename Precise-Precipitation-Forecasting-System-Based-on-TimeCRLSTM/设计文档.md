# sEMG数据收集上位机设计文档
## 数据通讯部分
### 采用两种通讯方式:串口通讯或WIFI通讯
1. 串口通讯
   - 波特率:从9600->3686400 可选框选择
   - 可以选择串口设备(使用可选框),默认为COM1(windows) /dev/ttyUSB0(linux)
2. WIFI通讯
   - 使用socket通讯
   - IP、端口均为输入框输入
   - 输入后使用socket进行连接通讯

### 设计要求
1. 使用Python+QT，也可以使用C+++QT或JavaFx实现，但是我没法给出任何的建议(没研究过C++和Java实现串口通讯)
2. 注意要实现通讯接口(TCP/IP以及串口通讯)

### 流程注意:
1. 先连接（串口可以使用`series`和`pyseries`库），WIFI使用`socket`库）
2. 连接成功后，先发送停止发送数据的指令，指令为`STOP_DATA`，随后发送获取sn的指令，获取设备sn，指令为`GET_SN`
3. 采集数据方法1——实时渲染
   1. 点击开始实时渲染按钮，发送开始收集数据的指令，获取数据帧，指令为`START_DATA`
   2. 获取数据帧后，解析数据帧，根据数据帧的格式，解析出数据，然后实时使用显示在界面上，可以使用`matplotlib`库、`plotly`库进行实时绘制，具体的情况随便大家选择，操作不间断
   3. 点击停止实时渲染按钮，发送停止收集数据的指令，指令为`STOP_DATA`
4. 采集数据方法2——采集并保存数据
   1. 在输入框中输入**收集数据的时间，单位是ms**,点击开始按钮，发送开始收集数据的指令，获取数据帧，指令为`START_DATA`
   2. 获取数据帧后，解析并保存数据帧，根据数据帧的格式，解析出数据，然后保存到文件中
   3. 当时间到达输入的时间后，自动发送停止收集数据的指令，指令为`STOP_DATA`，随后将文件保存到`.mat`格式的文件中，文件名为`sn_时间.mat`

### 指令设计
1. `STOP_DATA`：停止发送数据
2. `START_DATA`：开始发送数据
3. `GET_SN`：获取设备sn （注意，在使用这条指令前，请先发送`STOP_DATA`指令）
